# Internationalization (i18n)

Chiron includes built-in internationalization support for UI strings, making it easy to create documentation sites in any language.

## Version Support

### v1 (Current) - UI Strings
- ✅ **UI strings** - Search, navigation, buttons, etc.
- ✅ **Error messages** - Search errors, 404 pages, etc.
- ✅ **Accessibility labels** - ARIA labels, screen reader text
- ✅ **Cookie banner** - GDPR compliance messages
- ✅ **3 languages built-in** - English, Italian, French
- ✅ **Custom overrides** - Override any string per-site
- ✅ **Async loading** - Non-blocking, parallel locale loading

### v2 (Planned) - Full Content i18n
- ⏭️ Multiple language versions of content
- ⏭️ Language switcher UI
- ⏭️ Localized URLs (`/en/docs`, `/fr/docs`)
- ⏭️ Automatic language detection
- ⏭️ Per-page language override
- ⏭️ RTL support (Arabic, Hebrew)

## Quick Start

### 1. Set Your Language

Edit `chiron.config.yaml`:

```yaml
language:
  locale: it  # Change to your language code
```

### 2. Build Your Site

```bash
npm run build
```

That's it! All UI strings are now in Italian.

### 3. Custom Overrides (Optional)

Override specific strings:

```yaml
language:
  locale: it
  strings:
    search_placeholder: "Cerca qui..."
    theme_toggle: "Cambia tema"
```

---

## Configuration

Add i18n settings to `chiron.config.yaml`:

```yaml
# chiron.config.yaml

# Language settings
language:
  locale: en  # Default: en
  # Supported: en, it, fr, es, de, pt, nl, pl, ru, ja, zh, ko
  
  # Optional: Custom translations (override defaults)
  strings:
    search_placeholder: "Search docs..."
    theme_toggle: "Toggle dark mode"
    cookie_accept: "Accept"
    cookie_decline: "Decline"
```

### Custom Strings

Override any default translation:

```yaml
language:
  locale: en
  strings:
    search_placeholder: "Find what you need..."
    theme_toggle: "Switch theme"
    search_no_results: "Nothing found. Try different keywords."
    cookie_message: "We use cookies to improve your experience."
```

## Available String Keys

### Search

```yaml
search_placeholder: "Search documentation..."
search_no_results: "No results found"
search_loading: "Loading..."
search_error: "Search unavailable"
```

### Theme

```yaml
theme_toggle: "Toggle dark mode"
theme_light: "Light mode"
theme_dark: "Dark mode"
```

### Navigation

```yaml
nav_previous: "Previous"
nav_next: "Next"
nav_home: "Home"
nav_menu: "Menu"
nav_close: "Close"
```

### Cookie Banner

```yaml
cookie_message: "This website uses cookies to ensure you get the best experience."
cookie_accept: "Accept"
cookie_decline: "Decline"
cookie_learn_more: "Learn more"
```

### Accessibility

```yaml
aria_search: "Search documentation"
aria_menu: "Navigation menu"
aria_theme: "Toggle dark mode"
aria_close: "Close"
aria_expand: "Expand"
aria_collapse: "Collapse"
```

### Errors

```yaml
error_404_title: "Page Not Found"
error_404_message: "The page you're looking for doesn't exist."
error_404_home: "Go to homepage"
```

## Implementation

### In Templates

Strings are replaced with placeholders:

```html
<!-- Before -->
<button aria-label="Toggle dark mode">...</button>

<!-- After -->
<button aria-label="{{I18N_THEME_TOGGLE}}">...</button>
```

### In JavaScript

Strings are injected via config:

```javascript
// script.js
const i18n = window.CHIRON_I18N || {};

searchInput.placeholder = i18n.search_placeholder || 'Search documentation...';
```

## Contributing Translations

Want to add a language? Submit a PR with translations in `builder/i18n/locales/[lang].json`.

Example structure:

```json
{
  "search_placeholder": "Your translation...",
  "theme_toggle": "Your translation...",
  "cookie_accept": "Your translation...",
  ...
}
```

See `builder/i18n/locales/en.json` for the complete list of keys.
