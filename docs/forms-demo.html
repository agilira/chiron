Form Elements Showcase

This page demonstrates all form elements available in Chiron.

Using the FormField Component

Chiron provides a <FormField> component that simplifies form markup by automatically handling labels, helper text, error messages, IDs, and ARIA attributes.

Basic Example

Instead of writing verbose HTML:

<div class="form-field">
  <label for="email-input">Email Address</label>
  <input type="email" id="email-input" name="email" aria-describedby="email-helper" />
  <span class="form-helper-text" id="email-helper">We'll never share your email</span>
</div>

You can use:

<FormField label="Email Address" helper="We'll never share your email">
  <input type="email" name="email" />
</FormField>

  • Automatic ID generation: No need to manually manage IDs

  • Error handling: .has-error class applied automatically
  • Clean syntax: Less code, fewer errors

Form Validation

Browser Native Validation (Default) - All form components use HTML5 native validation by default:

<!-- Built-in validation attributes -->
<FormField label="Email *">
  <input type="email" required />
</FormField>

<FormField label="Age">
  <input type="number" min="18" max="100" />
</FormField>

<FormField label="Username">
  <input type="text" pattern="[a-zA-Z0-9]+" minlength="3" maxlength="20" />
</FormField>

Supported HTML5 Validation Attributes

  • required - Field must be filled
  • type="email" - Valid email format
  • type="url" - Valid URL format
  • type="number" - Numeric input with min, max, step
  • pattern - Custom regex validation
  • minlength / maxlength - String length constraints
  • min / max - Number/date range constraints

Custom Error Messages

Use the error attribute to display validation errors:

<FormField label="Password" error="Password must be at least 8 characters">
  <input type="password" minlength="8" />
</FormField>

<fieldset class="checkbox-group has-error" aria-describedby="checkbox-group-1763964426633-4-helper">
  <legend>Terms</legend>
  <div class="checkbox-group-options">
    <div class="checkbox-item">
      <input type="checkbox" id="checkbox-group-1763964426633-4-0" name="terms" value="agree" required />
      <label for="checkbox-group-1763964426633-4-0">I agree to the terms and conditions</label>
    </div>
  </div>
  <div class="form-error" id="checkbox-group-1763964426633-4-helper">You must accept the terms</div>
</fieldset>

Browser native validation provides:

  • Zero JavaScript required
  • Instant feedback
  • Accessible error messages
  • Works without JS enabled
  • Mobile-friendly UI
  • Automatic form submission prevention

Accessibility Features

The <FormField>, <CheckboxGroup>, and <RadioGroup> components automatically provide:

  • Proper Labels: Automatic for/id linking for screen readers
  • Helper Text: Connected via aria-describedby for accessibility
  • Error Messages: Clear error states with .has-error class and ARIA attributes
  • Focus Indicators: Visible focus rings for keyboard navigation (native browser)
  • Color Contrast: WCAG AA compliant color combinations (Carbon Design System)
  • Keyboard Support: Full keyboard accessibility (native HTML5)
  • HTML5 Validation: All native validation attributes preserved
  • XSS Protection: Automatic HTML escaping in labels, helpers, and errors
  • Fieldset/Legend: Proper semantic structure for checkbox/radio groups

Note: For complete documentation, see FormField, CheckboxGroup, and RadioGroup.
