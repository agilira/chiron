Buttons Showcase

Complete overview of all button variants in the Chiron Design System, based on Carbon Design System principles with accessibility at its core.

Design Principles

  • Accessibility First: All buttons meet WCAG 2.2 Level AA standards (minimum 44px touch target)
  • Clear Hierarchy: Visual weight indicates importance and action type
  • Consistent States: Hover, active, focus, and disabled states across all variants
  • Responsive: Touch-friendly on mobile, precise on desktop

Button Variants

Primary Buttons

Primary buttons for main actions. High visual prominence.

<button class="btn btn-primary">Primary Action</button>
<button class="btn btn-primary" disabled>Primary Disabled</button>

Secondary Buttons

Secondary actions with medium visual weight. Outlined style.

<button class="btn btn-secondary">Secondary Action</button>
<button class="btn btn-secondary" disabled>Secondary Disabled</button>

Tertiary Buttons

Tertiary actions with low visual weight. Ghost/minimal style.

<button class="btn btn-tertiary">Tertiary Action</button>
<button class="btn btn-tertiary" disabled>Tertiary Disabled</button>

Destructive Buttons

Critical actions like delete, remove, or irreversible operations.

<button class="btn btn-destructive">Delete</button>
<button class="btn btn-danger">Remove</button>
<button class="btn btn-destructive" disabled>Disabled</button>

Link Buttons

Text-style buttons for minimal visual weight.

<button class="btn btn-link">Link Button</button>
<button class="btn btn-link" disabled>Link Disabled</button>

Button Sizes

Small Buttons

Compact buttons for tight spaces (36px min-height).

<button class="btn btn-primary btn-sm">Small Primary</button>
<button class="btn btn-secondary btn-sm">Small Secondary</button>
<button class="btn btn-tertiary btn-sm">Small Tertiary</button>
<button class="btn btn-destructive btn-sm">Small Danger</button>

Default Buttons

Standard size for most use cases (44px min-height).

<button class="btn btn-primary">Default Primary</button>
<button class="btn btn-secondary">Default Secondary</button>
<button class="btn btn-tertiary">Default Tertiary</button>
<button class="btn btn-destructive">Default Danger</button>

Large Buttons

Prominent buttons for key actions (48px min-height).

<button class="btn btn-primary btn-lg">Large Primary</button>
<button class="btn btn-secondary btn-lg">Large Secondary</button>
<button class="btn btn-tertiary btn-lg">Large Tertiary</button>
<button class="btn btn-destructive btn-lg">Large Danger</button>

Button Icons

Buttons can include icons using inline SVG or icon sprites.

<button class="btn btn-primary">
  <svg>...</svg>
  Add Item
</button>
<button class="btn btn-secondary">
  Save
  <svg>...</svg>
</button>

Icon-Only Buttons

Compact buttons with only icons. Use aria-label for accessibility.

<button class="btn btn-primary" aria-label="Add">
  <svg>...</svg>
</button>

Button Groups

Horizontal groups of related buttons.

<div class="button-group">
  <button class="btn btn-secondary">Left</button>
  <button class="btn btn-secondary">Center</button>
  <button class="btn btn-secondary">Right</button>
</div>

<div class="button-group">
  <button class="btn btn-tertiary">Cancel</button>
  <button class="btn btn-secondary">Save Draft</button>
  <button class="btn btn-primary">Publish</button>
</div>

Full Width Buttons

Buttons that span the full width of their container.


<button class="btn btn-primary" style="width: 100%;">
  Full Width Primary
</button>

Accessibility Guidelines

Minimum Touch Target

All buttons meet WCAG 2.2 Level AA requirements:

  • Default buttons: 4444px minimum
  • Small buttons: 3636px (use sparingly)
  • Large buttons: 4848px for prominent actions

Keyboard Navigation

  • Tab: Navigate between buttons
  • Enter/Space: Activate button
  • Focus indicators: Visible 2px outline

Screen Readers

  • Use descriptive text
  • Add aria-label for icon-only buttons
  • Use aria-disabled for disabled state (handled automatically)

Color Contrast

All button variants meet WCAG AA standards:

  • Foreground/background contrast 4.5:1
  • Focus indicators contrast 3:1

Usage Guidelines

When to Use

  • Primary: Main action on a page/section (e.g., Submit, Save, Continue)
  • Secondary: Alternative actions (e.g., Cancel, Back, Preview)
  • Tertiary: Low-priority actions (e.g., Learn More, View Details)
  • Destructive: Irreversible actions (e.g., Delete, Remove)
  • Link: Navigation or minimal actions

Best Practices

Do:

  • Use one primary button per section
  • Keep button text concise (2-4 words)
  • Use icons to clarify actions
  • Provide visual feedback on interaction

Don't:

  • Use multiple primary buttons together
  • Mix button sizes in the same group
  • Use destructive buttons for non-critical actions
  • Rely solely on color to convey meaning

CSS Classes Reference

Class Description
.btn Base button styles (required)
.btn-primary Primary action button
.btn-secondary Secondary action button
.btn-tertiary Tertiary/ghost button
.btn-destructive Destructive action button
.btn-danger Alias for destructive
.btn-link Text/link style button
.btn-sm Small size (36px)
.btn-lg Large size (48px)
.button-group Container for button groups

Testing Checklist

Use this page to verify:

  • All variants render correctly in light mode
  • All variants render correctly in dark mode
  • Hover states are visually distinct
  • Active states provide clear feedback
  • Focus indicators are visible and meet contrast requirements
  • Disabled buttons are clearly distinguishable
  • Icons align properly with text
  • Button groups maintain consistent spacing
  • All buttons are keyboard accessible
  • Screen reader announces button purpose clearly
