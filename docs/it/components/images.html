Image Component

The <Image /> component provides powerful image capabilities that go far beyond what markdown syntax allows. It's perfect for responsive images, modern formats, captions, lazy loading, and full accessibility control.

Why Use It?

Markdown's ![alt](src) syntax is limited. You can't add:

  • Custom CSS classes or styling
  • Lazy loading and performance optimizations
  • Responsive images with srcset and sizes
  • Captions wrapped in semantic <figure> elements
  • Modern image formats (WebP, AVIF) with fallbacks
  • Link wrappers for clickable images
  • Alignment and layout control
  • Full accessibility attributes

The Image component solves all of these problems.

Basic Usage

<Image src="/images/photo.jpg" alt="A beautiful landscape" />

Output:

<img src="/images/photo.jpg" alt="A beautiful landscape" loading="lazy" decoding="async">

Note: The component automatically adds loading="lazy" and decoding="async" for optimal performance.

Live Demo

Here's a simple example in action:

Code:

<Image 
  src="https://picsum.photos/800/400" 
  alt="Random landscape photo from Lorem Picsum" 
  caption="A beautiful demo image with caption support"
  align="center"
  width="800"
  height="400"
/>

Clickable Image with Link

Code:

<Image 
  src="https://picsum.photos/600/400?random=1" 
  alt="Demo with link wrapper"
  href="https://picsum.photos/1200/800?random=1"
  caption="Click to view full size (opens larger demo image)"
  align="center"
  class="demo-image"
/>

Props Reference

Prop Type Default Description
src string required Image source URL (relative or absolute)
alt string "" Alternative text for accessibility
class string - CSS classes (also accepts className)
width string/number - Image width in pixels
height string/number - Image height in pixels
loading string "lazy" Native lazy loading ("lazy" or "eager")
srcset string - Responsive image sources
sizes string - Media conditions for responsive images
caption string - Caption text (wraps in <figure>)
figureClass string - CSS class for the <figure> element
href string - Wrap image in link with this URL
target string - Link target (_blank adds rel="noopener noreferrer")
align string - Alignment: "left", "center", "right"
webp string - WebP format source (generates <picture>)
avif string - AVIF format source (generates <picture>)
title string - Tooltip text on hover
role string - ARIA role (e.g., "presentation" for decorative images)

Examples

Custom Classes and Styling

<Image 
  src="/images/avatar.jpg" 
  alt="User avatar" 
  class="rounded-full shadow-lg border-4" 
  width="200" 
  height="200" 
/>

Lazy Loading Control

By default, images use loading="lazy". For above-the-fold images, use loading="eager":

<Image 
  src="/images/hero.jpg" 
  alt="Hero banner" 
  loading="eager" 
/>

Responsive Images

Use srcset and sizes for responsive images that adapt to screen size:

<Image 
  src="/images/photo-800.jpg" 
  alt="Responsive photo"
  srcset="/images/photo-400.jpg 400w, /images/photo-800.jpg 800w, /images/photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px"
/>

Figure with Caption

Wrap images in semantic <figure> elements with captions:

<Image 
  src="/images/sunset.jpg" 
  alt="Sunset over the ocean" 
  caption="A stunning sunset captured in Santorini, Greece" 
/>

Output:

<figure>
  <img src="/images/sunset.jpg" alt="Sunset over the ocean" loading="lazy" decoding="async">
  <figcaption>A stunning sunset captured in Santorini, Greece</figcaption>
</figure>

Alignment

Center, left-align, or right-align images:

<Image 
  src="/images/logo.png" 
  alt="Company logo" 
  align="center" 
/>

When used with caption, alignment is applied to the <figure>:

<Image 
  src="/images/diagram.png" 
  alt="Architecture diagram" 
  caption="System architecture overview"
  align="center" 
/>

Clickable Images

Wrap images in links for galleries or full-size previews:

<Image 
  src="/images/thumbnail.jpg" 
  alt="Gallery photo" 
  href="/images/full-size.jpg"
  caption="Click to view full size"
/>

Open in new tab:

<Image 
  src="/images/product.jpg" 
  alt="Product photo" 
  href="https://shop.example.com/product"
  target="_blank"
/>

Modern Image Formats (WebP, AVIF)

Use the <picture> element for modern formats with fallbacks:

<Image 
  src="/images/photo.jpg" 
  webp="/images/photo.webp"
  avif="/images/photo.avif"
  alt="High-quality photo" 
/>

Output (AVIF loads first for best compression):

<picture>
  <source srcset="/images/photo.avif" type="image/avif">
  <source srcset="/images/photo.webp" type="image/webp">
  <img src="/images/photo.jpg" alt="High-quality photo" loading="lazy" decoding="async">
</picture>

Decorative Images

For purely decorative images, use role="presentation" and empty alt:

<Image 
  src="/images/decoration.svg" 
  alt="" 
  role="presentation" 
/>

Accessibility Best Practices

  1. Always provide alt text for meaningful images

    <Image src="/photo.jpg" alt="Dog playing in the park" />

  2. Use empty alt="" for decorative images

    <Image src="/decoration.svg" alt="" role="presentation" />

  3. Use descriptive captions when context is important

    <Image src="/chart.png" alt="Sales chart" caption="Q4 2024 revenue increased 45%" />

  4. Specify dimensions to prevent layout shifts (better CLS scores)

    <Image src="/banner.jpg" alt="Banner" width="1200" height="400" />

Performance Features

The Image component automatically optimizes for performance:

  • Native lazy loading: Images below the fold load only when needed (loading="lazy")
  • Async decoding: Prevents blocking the main thread (decoding="async")
  • Modern formats: WebP and AVIF support reduces file sizes by 30-80%
  • Responsive images: Serve appropriately sized images for each device
  • Layout stability: Width and height prevent cumulative layout shift (CLS)

Browser Support

  • Lazy loading: All modern browsers (Chrome 76+, Firefox 75+, Safari 15.4+)
  • WebP: All modern browsers (94%+ global support)
  • AVIF: Chrome 85+, Firefox 93+, Safari 16+ (80%+ global support)
  • Picture element: All modern browsers (96%+ global support)

For older browsers, the component gracefully falls back to the standard <img> element.

FAQ

Q: Can I use both class and className?
A: They're aliases - use whichever you prefer. React developers typically use className.

Q: Should I use lazy loading for all images?
A: No. Use loading="eager" for above-the-fold images (hero banners, logos) to ensure they load immediately.

Q: What's the difference between WebP and AVIF?
A: AVIF offers 20-30% better compression than WebP. Use both for best results - browsers will pick the best format they support.

Q: Can I use external URLs?
A: Yes! The component supports relative paths, absolute URLs, protocol-relative URLs, and even data URLs.

Q: How do I style the caption?
A: Use CSS to target figcaption elements. You can also add classes to the figure with figureClass.

Q: What if I forget the src attribute?
A: The component will log a warning and return nothing. The src attribute is required.
